API

First, a bit of philosophy:

Java has most of the functionality a developer would want. Sometimes the API for that functionality is insane. For example, writing a file. In these cases, Gosu will typically offer a simpler alternative, often built in to the obvious Java class, via enhancements. Other times the API is fine and Gosu won't offer anything beyond that.

Gosu is not an attempt to reinvent the world. The core Java libraries are very solid and, in some cases, just lack a bit of simplicity or a bit of functionality, often due to a lack of features in Java.

Because of that, the API that Gosu provides is relatively small and somewhat sporadic, addressing corners of the standard Java API that we were unhappy with.
